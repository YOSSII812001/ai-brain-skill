# inbox/層のルール

- vault root直下のバッチ投入バッファ
- ユーザーがingest予定の素材を事前配置する
- AIは読み取り＋処理後の削除のみ（内容変更禁止）
- サブフォルダ不可（inbox/直下のみ）
- 対応形式: .md / .txt / .html / テキスト系全般
- バイナリ（.jpg/.png等）はスキップ
- **処理成功→必ず自動削除（Bash rmで実行 + Globで0件確認）**、失敗→inbox/に残留
- `/wiki-ingest-inbox` で一括処理を実行

## 削除の実行手順（必須）

ingest処理の最終ステップとして、以下を**必ず**実行すること:

1. `rm` コマンドで成功ファイルを個別に削除
2. `Glob` ツールで inbox/ を再スキャンし残留ファイル数を確認
3. 全成功なら「0 files remaining」であることを検証

**注意**: index.md更新・log.md記録・inbox削除は全てバッチ末尾で実行する。
これらのうち1つでも欠けた場合、ingest-inboxは未完了とみなす。
